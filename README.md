# Forge

Local dev process orchestration daemon with web dashboard.

Forge runs a background daemon that manages your dev processes (servers, workers, watchers) across multiple projects. Each project gets allocated ports, shared services (Postgres, Redis, Mongo) are started on demand, and a web dashboard at `localhost:2525` shows live terminal output for every process.

## Requirements

- macOS (Linux support planned)
- Node.js ≥ 20
- Docker Desktop — required for shared services (Mongo, Redis)
- Xcode Command Line Tools (`xcode-select --install`) — required by `node-pty` for terminal emulation

## Install

```bash
npm install -g @brutalsystems/forge
forge install
```

`forge install` registers the daemon as a launchd agent so it starts automatically on login.

## Quick start

```bash
# 1. In your project directory, scaffold a config
cd ~/projects/my-app
forge init

# 2. Edit .forge/config.json to describe your processes
#    (see Configuration below)

# 3. Register the project with the daemon
forge add

# 4. Start all processes
forge up my-app

# 5. Open the web dashboard
forge open
```

## Configuration

`forge init` creates `.forge/config.json` in the current directory. Edit it to describe your project's processes and which shared services it needs.

### Full example

```json
{
  "name": "my-app",
  "envFile": ".env.forge",
  "processes": [
    {
      "name": "api",
      "command": "npm start",
      "cwd": ".",
      "ports": [3000, 3001, 3002],
      "portEnv": "PORT"
    },
    {
      "name": "worker",
      "command": "node worker.js",
      "cwd": ".",
      "ports": [],
      "portEnv": null
    },
    {
      "name": "ui",
      "command": "npm run dev",
      "cwd": "packages/ui",
      "ports": [5173, 5174, 5175],
      "portEnv": "VITE_PORT"
    }
  ],
  "services": {
    "mongo": {
      "db": "my-app-dev",
      "env": "MONGODB_URL"
    },
    "redis": {
      "env": "REDIS_URL",
      "prefix": "my-app:",
      "prefixEnv": "REDIS_PREFIX"
    }
  }
}
```

### Process fields

| Field | Description |
|---|---|
| `name` | Unique process name within the project |
| `command` | Shell command to run |
| `cwd` | Working directory, relative to the project root |
| `ports` | Candidate ports to try in order. Forge picks the first available. |
| `portEnv` | Environment variable that receives the allocated port at startup |

### Port allocation

When you run `forge add`, the daemon picks the first available port from each process's `ports` list and reserves it. On startup, Forge injects `portEnv=<port>` directly into the spawned process environment — you don't need to set it yourself.

### Service fields

#### MongoDB

```json
"mongo": {
  "db": "my-app-dev",
  "env": "MONGODB_URL"
}
```

| Field | Description |
|---|---|
| `db` | Database name. Defaults to the project name if omitted. |
| `env` | Env var written to `.env.forge` with the connection string (`mongodb://localhost:27017/<db>`). |

#### Redis

```json
"redis": {
  "env": "REDIS_URL",
  "prefix": "my-app:",
  "prefixEnv": "REDIS_PREFIX"
}
```

| Field | Description |
|---|---|
| `env` | Env var written to `.env.forge` with the connection string (`redis://localhost:6379/<db>`). Each project gets its own Redis database number (1–63). |
| `prefix` | Optional key prefix string for namespacing Redis keys. |
| `prefixEnv` | Env var that receives the prefix value. |

Forge starts a single shared `forge-redis` (Redis 7) and `forge-mongo` (Mongo 7) Docker container on demand and provisions each project into an isolated database.

### envFile

`forge add` (and `forge sync`) write a `.env.forge` file at the project root with the resolved service URLs:

```bash
# generated by forge — do not edit
MONGODB_URL=mongodb://localhost:27017/my-app-dev
REDIS_URL=redis://localhost:6379/3
REDIS_PREFIX=my-app:
```

Load this file in your process startup (e.g. `dotenv.config({ path: '.env.forge' })`). Port env vars (`PORT`, etc.) are injected directly at spawn time and do not appear in `.env.forge`.

Set `"envFile": false` to skip writing the file entirely.

## Commands

| Command | Description |
|---|---|
| `forge install` | Register daemon as launchd agent and start it |
| `forge uninstall` | Stop daemon and remove launchd agent |
| `forge init` | Scaffold a `.forge/config.json` in the current directory |
| `forge add` | Register the current project with the daemon |
| `forge sync` | Re-read `.forge/config.json` and update allocations (run after editing config) |
| `forge remove <project>` | Unregister a project |
| `forge up [project]` | Start all processes for a project |
| `forge down [project]` | Stop all processes for a project |
| `forge open` | Open the web dashboard in your browser |
| `forge status` | Show status of all registered projects |
| `forge services` | Show shared service health (Mongo, Redis) |
| `forge env <project>` | Print allocated env vars for a project |
| `forge extend <path>` | Merge processes from another project into the current config |

### forge extend

`forge extend` is for monorepo setups or when you want to run processes from multiple repos under a single project. It reads the target directory's config (or infers processes from `package.json` / `pyproject.toml` / `Makefile`) and appends them to the current `.forge/config.json`, prefixed with the source project name.

```bash
cd ~/projects/my-app
forge extend ../my-worker
```

This adds processes like `my-worker:start` to `my-app`'s config, with `cwd` set relative to `my-app`'s root. Run `forge sync` afterward to apply if already registered.

## Dashboard

After `forge install`, the dashboard is available at `http://localhost:2525`. It shows:

- All registered projects and their process status
- Live terminal output per process (via xterm.js over WebSocket)
- Shared service health (Mongo, Redis)
- Per-process start / stop / restart controls
- Drag-to-reorder process panels (order persists per project)

## Troubleshooting

**Daemon didn't start**
```bash
cat ~/.forge/daemon.error.log
```

**Check daemon status**
```bash
launchctl list | grep forge
```

**node-pty fails to install**
Xcode Command Line Tools are required. Run:
```bash
xcode-select --install
```

**Port conflict**
Add more candidate ports to the `ports` array for the affected process, then run `forge sync`.

**Service won't start**
Forge uses Docker to run Mongo and Redis. Make sure Docker Desktop is running:
```bash
docker ps
forge services
```

**Reset a project's allocations**
```bash
forge remove my-app
forge add
```
